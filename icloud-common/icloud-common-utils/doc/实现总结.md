## 功能特性

### 1. 注解式快速开发

只需在Controller方法上添加 `@TableAction` 注解，即可实现：
- 分页查询
- 新增数据
- 更新数据
- 删除数据

无需编写重复的CRUD代码。

### 2. 自动表信息扫描

系统启动时自动扫描：
- 所有继承 `BaseMapper` 的Bean
- 对应的实体类
- 对应的Condition类（如果存在）

### 3. 事件驱动架构

每次增删改操作完成后，自动发布 `TableChangeEvent` 事件：
- 支持监听所有表的变更
- 支持按表名过滤
- 支持按操作类型过滤
- 事件中包含完整的数据信息

### 4. 灵活的事件订阅

提供了多种事件订阅方式：

```java
// 监听所有事件
@EventListener
public void handleEvent(TableChangeEvent event) { }

// 监听特定表
@EventListener(condition = "#event.tableName == 'sys_user'")
public void handleUserChange(TableChangeEvent event) { }

// 监听特定操作
@EventListener(condition = "#event.action == T(...ActionType).INSERT")
public void handleInsert(TableChangeEvent event) { }
```

## 使用示例

### 基本用法

```java
@RestController
@RequestMapping("/user")
public class UserController {

    @TableAction(tableName = "sys_user", action = ActionType.INSERT)
    @PostMapping("/insert")
    public Response<Object> insert(@RequestBody Object data) {
        return null; // 由切面处理
    }
}
```

### 事件监听

```java
@Component
public class UserChangeListener {

    @EventListener(condition = "#event.tableName == 'sys_user'")
    public void onUserChange(TableChangeEvent event) {
        if (event.isInsert()) {
            log.info("新用户创建: ID={}", event.getDataId());
            // 发送欢迎邮件
            // 初始化用户配置
            // 等等...
        }
    }
}
```

## 技术要点

1. **AOP切面**: 使用 `@Around` 拦截注解方法
2. **反射机制**: 动态获取表信息和执行数据库操作
3. **Spring事件**: 使用 `ApplicationEventPublisher` 发布事件
4. **自动配置**: 通过 `@ComponentScan` 自动注册所有组件
5. **JSON处理**: 使用 Fastjson2 进行对象序列化/反序列化

## 注意事项

### IDE编译错误

如果IDE显示编译错误（特别是AspectJ相关的），请尝试：
1. 重新加载Maven项目
2. 清理并重新构建项目
3. 重启IDE

这些错误通常是IDE缓存问题，实际编译应该没问题。

### 实体类命名规范

为了自动扫描正常工作，建议遵循以下命名规范：
- 实体类：`SysUser` (驼峰命名)
- Mapper Bean：`sysUserMapper` (首字母小写 + "Mapper"后缀)
- Condition类：`SysUserCondition` (实体类名 + "Condition"后缀)
- 包路径：`.entity.` 和 `.condition.` (扫描器会自动替换)

### 自定义扫描路径

如果实体类不在默认路径下，可以修改 `TableInfoScanner` 中的 `basePackages` 数组，或者手动注册表信息。
