# 事件机制升级说明

## 概述

事件机制已从 Spring ApplicationEvent 升级为基于**三级路由**的订阅者模式，参考了 rpa-admin 项目的实现。

## 核心改进

### 1. 三级路由机制

```
TABLE_OP : INSERT/UPDATE/DELETE : 表名
  r1          r2                  r3
```

- **r1（第一级）**: `TABLE_OP` - 固定前缀，避免与用户自定义事件冲突
- **r2（第二级）**: 操作类型（INSERT/UPDATE/DELETE）
- **r3（第三级）**: 表名或业务标识

### 2. 自动注册与发现

通过 `@RouteType` 注解和 `BaseFactory` 实现订阅者的自动注册和路由查找：

```java
@Component
@RouteType(
    r1 = TableOperationEventConstants.R1_TABLE_OPERATION,
    r2 = TableOperationEventConstants.R2_INSERT,
    r3 = "sys_user"
)
public class SysUserInsertSubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        // 处理事件
    }
}
```

## 新增文件

### 核心类

1. **TableOperationEventConstants.java** - 事件常量类
   - 定义固定前缀 `TABLE_OP`
   - 提供便捷的事件类型构建方法

2. **TableOperationEventMsg.java** - 事件消息类
   - 包含完整的事件信息
   - 替代原来的 TableChangeEvent

3. **TableOperationEventSubscriber.java** - 订阅者接口
   - 替代原来的 Spring EventListener

4. **TableOperationEventSubscriberFactory.java** - 订阅者工厂
   - 继承 BaseFactory，实现三级路由查找

5. **TableOperationEventPublisher.java** - 事件发布器
   - 根据三级路由自动找到订阅者
   - 支持并行执行多个订阅者

### 示例代码

6. **SysUserInsertEventSubscriber.java** - 用户插入事件订阅者示例
7. **SysUserUpdateEventSubscriber.java** - 用户更新事件订阅者示例
8. **GlobalInsertEventSubscriber.java** - 全局插入事件订阅者示例

## 删除文件

1. ~~TableChangeEvent.java~~ - 旧的Spring事件类
2. ~~TableChangeEventExampleListener.java~~ - 旧的事件监听器示例

## 使用对比

### 旧方式（Spring EventListener）

```java
@Component
public class OldListener {
    @EventListener
    public void handleEvent(TableChangeEvent event) {
        if (event.isInsert() && "sys_user".equals(event.getTableName())) {
            // 处理逻辑
        }
    }
}
```

**缺点**：
- 需要手动判断事件类型
- 所有订阅者都会收到所有事件
- 无法精确路由

### 新方式（三级路由）

```java
@Component
@RouteType(
    r1 = TableOperationEventConstants.R1_TABLE_OPERATION,
    r2 = TableOperationEventConstants.R2_INSERT,
    r3 = "sys_user"
)
public class NewSubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        // 直接处理，无需判断
    }
}
```

**优点**：
- ✅ 精确路由，只调用匹配的订阅者
- ✅ 自动注册，无需手动判断
- ✅ 固定前缀，避免与用户自定义事件冲突
- ✅ 支持一个事件多个订阅者
- ✅ 并行执行，提升性能

## 迁移指南

如果你之前使用了旧的事件机制，需要按以下步骤迁移：

### 1. 替换导入

```java
// 旧
import cn.isqing.icloud.common.utils.event.TableChangeEvent;
import org.springframework.context.event.EventListener;

// 新
import cn.isqing.icloud.common.utils.event.TableOperationEventSubscriber;
import cn.isqing.icloud.common.utils.event.TableOperationEventMsg;
import cn.isqing.icloud.common.utils.annotation.RouteType;
import cn.isqing.icloud.common.utils.constants.TableOperationEventConstants;
```

### 2. 实现接口

```java
// 旧
@Component
public class MyListener {
    @EventListener
    public void handleEvent(TableChangeEvent event) {
        // 处理逻辑
    }
}

// 新
@Component
@RouteType(
    r1 = TableOperationEventConstants.R1_TABLE_OPERATION,
    r2 = TableOperationEventConstants.R2_INSERT,  // 或其他操作
    r3 = "your_table_name"
)
public class MySubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        // 处理逻辑
    }
}
```

### 3. 访问事件数据

```java
// 旧
event.getTableName()
event.getAction()
event.getDataId()
event.getData()
event.isSuccess()

// 新
eventMsg.getTableName()
eventMsg.getOperation()
eventMsg.getDataId()
eventMsg.getData()  // JSON字符串
eventMsg.getSuccess()
```

## 高级用法

### 1. 一个事件多个订阅者

可以创建多个订阅者监听同一个事件：

```java
// 订阅者1：发送欢迎邮件
@Component
@RouteType(r1 = TABLE_OP, r2 = INSERT, r3 = "sys_user")
public class WelcomeEmailSubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        sendEmail(eventMsg.getDataId());
    }
}

// 订阅者2：初始化配置
@Component
@RouteType(r1 = TABLE_OP, r2 = INSERT, r3 = "sys_user")
public class InitConfigSubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        initConfig(eventMsg.getDataId());
    }
}
```

两个订阅者会**并行执行**。

### 2. 全局订阅

第三级留空可以匹配所有表：

```java
@Component
@RouteType(
    r1 = TableOperationEventConstants.R1_TABLE_OPERATION,
    r2 = TableOperationEventConstants.R2_INSERT,
    r3 = ""  // 匹配所有表
)
public class GlobalInsertSubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        log.info("任意表插入: {}", eventMsg.getTableName());
    }
}
```

### 3. 错误处理

订阅者中的异常不会影响其他订阅者：

```java
@Component
@RouteType(r1 = TABLE_OP, r2 = INSERT, r3 = "sys_user")
public class SafeSubscriber implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        try {
            // 你的逻辑
        } catch (Exception e) {
            log.error("处理失败", e);
            // 不会影响到其他订阅者
        }
    }
}
```

## 技术实现

### BaseFactory 工作原理

1. **启动时扫描**：`@PostConstruct` 扫描所有实现了 `TableOperationEventSubscriber` 的Bean
2. **提取路由**：从 `@RouteType` 注解中提取 r1、r2、r3
3. **构建Key**：格式为 `r1:r2:r3`
4. **注册到Map**：`Map<String, List<Subscriber>>`
5. **运行时查找**：根据事件类型快速定位订阅者列表

### 事件发布流程

```
TableOperationServiceImpl
    ↓ 调用
TableOperationEventPublisher.publishInsertEvent()
    ↓ 构建
TableOperationEventMsg
    ↓ 查找
TableOperationEventSubscriberFactory.get(r1, r2, r3)
    ↓ 返回
List<TableOperationEventSubscriber>
    ↓ 并行执行
subscriber.onEvent(eventMsg)
```

## 注意事项

1. **固定前缀**：所有表操作事件都以 `TABLE_OP` 开头，避免与用户自定义事件冲突
2. **并行执行**：订阅者默认并行执行，注意线程安全
3. **异常隔离**：单个订阅者异常不会影响其他订阅者
4. **空值匹配**：r3为空时匹配所有表
5. **必须加注解**：订阅者必须添加 `@Component` 和 `@RouteType` 注解

## 总结

新的三级路由事件机制提供了：
- ✅ 更精确的路由
- ✅ 更好的性能（并行执行）
- ✅ 更清晰的职责分离
- ✅ 避免命名冲突（固定前缀）
- ✅ 更灵活的订阅方式

建议所有新功能都使用新的事件机制。
