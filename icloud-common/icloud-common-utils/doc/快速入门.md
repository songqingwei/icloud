# 快速入门 - 5分钟上手注解式增删改查

## 第一步：了解核心概念

本工具通过 `@TableAction` 注解实现快速增删改查，无需编写重复的CRUD代码。

## 第二步：创建你的实体和Mapper

假设你有一个用户表，需要创建：

### 1. 实体类
```java
package cn.isqing.icloud.dal.entity;

import lombok.Data;

@Data
public class SysUser {
    private Long id;
    private String username;
    private String email;
    private Integer status;
}
```

### 2. Mapper接口
```java

import cn.isqing.icloud.common.utils.dao.BaseMapper;
import cn.isqing.icloud.dal.entity.SysUser;
import org.springframework.stereotype.Repository;

@Repository
public interface SysUserMapper extends BaseMapper<SysUser> {
}
```

### 3. Condition类（用于查询）
```java
package cn.isqing.icloud.dal.condition;

import cn.isqing.icloud.common.utils.dto.BaseCondition;
import lombok.Data;
import lombok.EqualsAndHashCode;

@Data
@EqualsAndHashCode(callSuper = true)
public class SysUserCondition extends BaseCondition {
    private String username;
    private Integer status;
}
```

## 第五步：订阅事件（可选）

如果你想在用户创建后执行一些额外操作（如发送欢迎邮件），需要创建一个事件订阅者：

```java
package cn.isqing.icloud.listener;

import cn.isqing.icloud.common.utils.annotation.RouteType;
import cn.isqing.icloud.common.utils.constants.TableOperationEventConstants;
import cn.isqing.icloud.common.utils.event.TableOperationEventMsg;
import cn.isqing.icloud.common.utils.event.TableOperationEventSubscriber;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;

@Slf4j
@Component
@RouteType(
    r1 = TableOperationEventConstants.R1_TABLE_OPERATION,  // TABLE_OP
    r2 = TableOperationEventConstants.R2_INSERT,            // INSERT
    r3 = "sys_user"                                          // sys_user表
)
public class UserInsertEventListener implements TableOperationEventSubscriber {

    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        log.info("新用户创建，ID: {}", eventMsg.getDataId());
        
        if (eventMsg.getSuccess()) {
            // 发送欢迎邮件
            sendWelcomeEmail(eventMsg.getDataId());
            
            // 初始化用户配置
            initUserConfig(eventMsg.getDataId());
        }
    }
    
    private void sendWelcomeEmail(Long userId) {
        // 实现发送邮件逻辑
    }
    
    private void initUserConfig(Long userId) {
        // 实现初始化配置逻辑
    }
}
```

### 三级路由说明

事件订阅采用**三级路由机制**：

```
TABLE_OP : INSERT/UPDATE/DELETE : 表名
  r1          r2                  r3
```

- **r1**: `TABLE_OP` - 固定前缀，避免与用户自定义事件冲突
- **r2**: 操作类型（INSERT/UPDATE/DELETE）
- **r3**: 表名或业务标识

### 更多订阅示例

#### 1. 订阅所有表的插入操作

```java
@Component
@RouteType(
    r1 = TableOperationEventConstants.R1_TABLE_OPERATION,
    r2 = TableOperationEventConstants.R2_INSERT,
    r3 = ""  // 空表示匹配所有表
)
public class GlobalInsertListener implements TableOperationEventSubscriber {
    @Override
    public void onEvent(TableOperationEventMsg eventMsg) {
        log.info("表 {} 新增数据", eventMsg.getTableName());
    }
}
```

#### 2. 订阅用户表的所有操作

创建三个订阅者分别处理INSERT、UPDATE、DELETE。

### 3. 手动注册表信息

如果自动扫描不工作，可以手动注册：

```java
@Autowired
private TableInfoScanner tableInfoScanner;

@PostConstruct
public void init() {
    TableInfoDto tableInfo = new TableInfoDto();
    tableInfo.setEntityClass(SysUser.class);
    tableInfo.setMapper(sysUserMapper);
    tableInfo.setConditionClass(SysUserCondition.class);
    
    tableInfoScanner.registerTableInfo("sys_user", tableInfo);
}
```

## 常见问题

### Q: 为什么我的注解没有生效？

A: 检查以下几点：
1. 确保添加了 `spring-boot-starter-aop` 依赖
2. 确保Mapper继承了 `BaseMapper`
3. 确保表名正确（不带反引号）
4. 查看日志，确认表信息已扫描

### Q: 如何查看扫描了哪些表？

A: 启动应用时查看日志，会输出类似：
```
注册表信息: sys_user -> SysUser
表信息映射初始化完成，共加载10个表
```

### Q: 事件处理失败会影响主流程吗？

A: 不会。事件处理异常已被捕获，不会影响增删改操作。

### Q: 支持事务吗？

A: 支持。如果需要事务，可以在Service层添加 `@Transactional` 注解。
